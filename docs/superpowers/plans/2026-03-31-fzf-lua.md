# Fzf-Lua Configuration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add `ibhagwan/fzf-lua` to this Neovim config with project-navigation-first defaults plus practical LSP and Git pickers.

**Architecture:** Keep plugin declaration in a dedicated `lua/plugins/fzf.lua` file, register it from `lua/plugins/init.lua`, and define all user-facing mappings in `lua/config/keymaps.lua` to match the repo's current structure. Use lazy loading and a compact `opts` table so the setup stays focused and easy to extend.

**Tech Stack:** Lua, Neovim, lazy.nvim-style plugin specs, `ibhagwan/fzf-lua`, optional `nvim-web-devicons`

Add a plugin table for `ibhagwan/fzf-lua` with:

```lua
return {
    "ibhagwan/fzf-lua",
    cmd = "FzfLua",
    keys = {
        "<leader>ff",
        "<leader>fg",
        "<leader>fb",
    },
    dependencies = {
        { "nvim-tree/nvim-web-devicons", enabled = vim.g.have_nerd_font ~= false },
    },
    opts = {
        winopts = {
            border = "rounded",
            preview = {
                border = "rounded",
                layout = "vertical",
                vertical = "down:55%",
            },
        },
        files = {
            hidden = true,
            no_ignore = false,
            cwd_prompt = false,
            rg_opts = "--color=never --files --hidden -g !.git",
        },
        grep = {
            hidden = true,
            rg_opts = "--column --line-number --no-heading --color=always --smart-case --hidden -g !.git -e",
        },
    },
}
```

### Task 3: Add user-facing keymaps

**Files:**
- Modify: `lua/config/keymaps.lua`

- [ ] **Step 1: Preserve existing `dropbar` mappings**

Do not edit or remove the current `dropbar` keymaps.

- [ ] **Step 2: Add a small local helper**

Add a helper so each map can call a picker lazily:

```lua
local function fzf(picker, opts)
    return function()
        require("fzf-lua")[picker](opts or {})
    end
end
```

- [ ] **Step 3: Add project-navigation mappings**

Implement:
